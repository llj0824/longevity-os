OpenClaw Demo Readiness

The live Telegram entry point is simple, but the internal skill graph is much richer than the current demo artifact made obvious.

Longevity OS does not expose nine separate bot commands. It exposes one skill surface through /longevity and /taiyiyuan, then the Imperial Physician coordinator classifies the request and dispatches it to one or more specialist agents. The HTML now needs to teach that architecture clearly, because it is part of why better seed data matters: the demo value comes from showing the coordinator choose the right lane and synthesize the result.

Skill Structure

Command surface versus internal agent graph

The skill exposes a small command surface and a large internal routing graph. That is why the demo wants more than a single canned report: the strongest proof is that the coordinator can route logging, querying, reporting, and trial workflows to the correct specialist lane.

Ranked Changes

Seed-data changes that would better demonstrate the coordinator plus specialist graph

Now that the command structure is explicit, the seed-data recommendation becomes clearer: we should optimize for moments where the Imperial Physician visibly chooses the right specialist path or combines several of them.

Rank Change Why it helps the architecture show up
1 Seed one cleaner flagship experiment and make it the default report-era story. The coordinator can then route from report to trial-design to safety-review without the narrative getting bogged down in confounders.
2 Seed one mixed-intent day such as food plus workout plus sleep drop plus supplement change. This creates a natural reason for the Imperial Physician to fan out across multiple departments in one response.
3 Add one biomarker-triggered coaching moment near the demo week. This shows the coordinator can bridge 验方科, 本草科, and 报告科 instead of only generating trend summaries.
4 Add a short setback-and-recovery arc with a clear cross-module explanation. This gives the report lane a real “why did this happen?” reveal rather than a smooth improvement story.
